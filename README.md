### Задание 1

`Опишите основные преимущества использования масштабирования методами:`

1. `Активный master-сервер и пассивный репликационный slave-сервер`
Это классическая master-slave (репликация «один к одному») с пассивным резервом.

Преимущества:

Отказоустойчивость: при выходе из строя master-сервера slave можно вручную (или автоматически через оркестратор) перевести в режим master и продолжить работу.
Резервное копирование без нагрузки на master: бэкапы можно делать с slave, не мешая основной работе.
Простота настройки и управления: всего два узла, понятная архитектура.
Чтение с реплики (опционально): если slave не используется только как пассивный резерв, его можно подключить для обработки запросов SELECT, снижая нагрузку на master.

ажно: пассивный slave обычно не обслуживает запросы, а держится «в готовности». Это отличает его от активного slave в схеме «master + несколько slaves». 

2. `Master-сервер и несколько slave-серверов`
Это масштабируемая асинхронная репликация «один ко многим».

Преимущества:

Масштабирование чтения: все запросы SELECT можно распределять между slave-серверами (например, через балансировщик), что значительно снижает нагрузку на master.
Гибкость размещения: slave можно размещать в разных дата-центрах или регионах для снижения задержек у пользователей.
Изолированные рабочие нагрузки: один slave — для веб-приложения, другой — для аналитики, третий — для бэкапов.
Повышенная отказоустойчивость: даже при потере одного или нескольких slave система продолжает работать.

Ограничение: запись возможна только на master, поэтому масштабирование записи таким способом не решается.
